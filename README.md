## 📝 Descripción

Este repositorio contiene el **informe de la exposición grupal** correspondiente al **segundo corte** de la asignatura **Ingeniería de Requisitos**, en el que se abordaron distintas herramientas CASE —**StarUML, Umple Online, Eclipse Papyrus, BOUML, Modelio y Visual Paradigm**— utilizadas para el modelado UML y la generación automática de código (ingeniería directa, inversa y bidireccional).

El objetivo del trabajo es comparar estas herramientas según criterios técnicos comunes (modelado, generación de código, ingeniería inversa, facilidad de uso, entre otros) y documentar la práctica realizada con cada una.

## ✅ Cómo revisar este trabajo

Para facilitar la revisión y calificación, el informe cubre los siguientes puntos. Cada uno se puede verificar abriendo el PDF o el `.md` principal:

- [x] Definición de conceptos **MDD, MDE y MDA**.
- [x] Explicación de transformaciones **M2M** y **M2T**.
- [x] Descripción y prueba práctica de **cada herramienta CASE** (StarUML, Umple Online, Eclipse Papyrus + Acceleo, BOUML, Modelio, Visual Paradigm).
- [x] Evidencia visual (capturas de pantalla) de la ejecución de cada herramienta, disponible en `imgenes_informe_pdf`.
- [x] Explicación de **ingeniería directa, inversa y bidireccional** aplicada en las prácticas.
- [x] Conclusiones y comparación final entre las herramientas evaluadas.

Si algo no se visualiza correctamente en GitHub, se recomienda **descargar el PDF** para la versión con el formato completo (tablas, imágenes y diagramas).

## 🧩 Temas cubiertos en el informe

- Conceptos de **MDD (Model-Driven Development)**, **MDE (Model-Driven Engineering)** y **MDA (Model-Driven Architecture)**.
- Transformaciones **M2M (Model to Model)** y **M2T (Model to Text)**.
- Comparación de herramientas CASE:
  - StarUML
  - Umple Online
  - Eclipse Papyrus + Acceleo
  - BOUML
  - Modelio
  - Visual Paradigm
- Ingeniería directa.
- Ingeniería inversa.
- Ingeniería bidireccional.
- Prácticas de generación automática de código a partir de diagramas UML.
